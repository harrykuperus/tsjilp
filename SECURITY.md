# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in Tsjilp, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email: security@tsjilp.me

Please include:

- A description of the vulnerability
- Steps to reproduce
- The potential impact
- Any suggested fix if you have one

## Response timeline

- **Acknowledgment** within 48 hours
- **Initial assessment** within 5 business days
- **Fix or mitigation plan** communicated as soon as possible

## Responsible disclosure

We ask that you:

- Give us reasonable time to fix the issue before disclosing it publicly
- Do not access or modify other users' data
- Do not degrade the service (no DDoS, no brute force)
- Report in good faith

We will:

- Not take legal action against good-faith reporters
- Credit you in the fix announcement (unless you prefer to remain anonymous)
- Work with you to understand and resolve the issue

## Supported versions

Only the latest release is supported for security updates.

## Scope

Security issues include:

- Authentication bypasses
- SQL injection
- Cross-site scripting (XSS)
- Cross-site request forgery (CSRF)
- Information disclosure of private user data
- API key or credential exposure
- Any issue that compromises user privacy
